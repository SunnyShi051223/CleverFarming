# 🌾 农情速递新闻个性化推送功能实现报告

## 一、功能概述

✅ **已实现个性化推送功能**

- **推送平台**: `farming_alerts` 表
- **区分方式**: 通过 `alert_type` 字段区分新闻和预警
  - `alert_type = 'news'`: 农业资讯新闻
  - `alert_type = 'alert'`: 气象预警和灾害预警
- **细分分类**: 通过 `alert_subtype` 字段进一步分类

## 二、系统架构

### 2.1 推荐引擎流程

```
┌─────────────────────────────────────────────────────────────┐
│                    推荐引擎运行流程                          │
├─────────────────────────────────────────────────────────────┤
│ 1. 获取所有用户（id, username, city, crop_type）            │
│ 2. 获取近期新闻池（最近30天的资讯，50条）                   │
│ 3. 为每个用户计算推荐得分：                                 │
│    - 显式兴趣匹配（作物直接命中）+5.0分                     │
│    - 隐式兴趣挖掘（相关病害/农药）+3.0分                    │
│    - 地区属性加持（所在城市）+2.0分                         │
│    - 时效性衰减（时间加权）                                 │
│ 4. 推送最高分新闻到 farming_alerts 表                      │
│    - alert_type: news/alert                                │
│    - alert_subtype: 分类细分                                │
└─────────────────────────────────────────────────────────────┘
```

## 五、核心算法

### 5.1 双塔推荐模型

```
得分 = 显式兴趣 + 隐式兴趣 + 地区加权 + 时效性衰减

其中：
- 显式兴趣: crop in title_entities → +5.0
- 隐式兴趣: implicit_word in title_entities → +3.0
- 地区加权: city in title_entities → +2.0
- 时效性: e^(-0.1 * days_old) (保底 0.1)
```
