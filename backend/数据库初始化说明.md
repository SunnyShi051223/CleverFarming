# 数据库一键初始化说明

## 🚀 快速开始

### 方法一：使用 Python 脚本（推荐）

1. **确保已安装依赖**
```bash
pip install pymysql bcrypt
```

2. **配置数据库连接**

编辑 `backend/config.py` 文件，设置数据库连接信息：
```python
MYSQL_HOST = 'localhost'
MYSQL_USER = 'root'
MYSQL_PASSWORD = 'your_password'  # 修改为你的MySQL密码
MYSQL_DB = 'farming_system'
MYSQL_PORT = 3306
```

3. **运行初始化脚本**
```bash
cd backend
python init_db.py
```

### 方法二：使用 SQL 文件

如果你更喜欢使用SQL文件：
```bash
mysql -u root -p < update_database.sql
```

## ⚠️ 注意事项

### 1. 数据库已存在
如果数据库已存在，脚本会：
- 保留现有表结构
- 不会删除已有数据
- 只添加缺失的表和字段

### 2. 重新初始化
如果需要完全重新初始化（会删除所有数据）：

```sql
DROP DATABASE IF EXISTS farming_system;
```

然后重新运行 `init_db.py`

### 3. 权限问题
确保MySQL用户有以下权限：
- CREATE DATABASE
- CREATE TABLE
- INSERT
- SELECT
- UPDATE
- DELETE

### 4. 字符集问题
所有表使用 `utf8mb4` 字符集，支持中文和emoji表情。

## 🛠️ 故障排查

### 问题1：连接失败
```
pymysql.err.OperationalError: (2003, "Can't connect to MySQL server")
```

**解决方案：**
- 检查MySQL服务是否启动
- 确认 `config.py` 中的连接信息正确
- 检查防火墙设置

### 问题2：权限不足
```
pymysql.err.OperationalError: (1044, "Access denied for user")
```

**解决方案：**
```sql
GRANT ALL PRIVILEGES ON farming_system.* TO 'root'@'localhost';
FLUSH PRIVILEGES;
```

### 问题3：表已存在
```
Table 'users' already exists
```

**说明：** 这是正常的，脚本使用 `IF NOT EXISTS`，不会覆盖现有表。

### 问题4：外键约束失败
```
Cannot add foreign key constraint
```

**解决方案：**
- 确保按顺序创建表（users表必须先创建）
- 检查引用的表和字段是否存在
